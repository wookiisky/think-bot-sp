# Think Bot 产品功能说明

## 1. 文档范围

本文档只描述当前已经实现的产品能力、用户可见行为和交互规则，不展开代码层实现细节。

产品形态：

- Chrome Manifest V3 扩展
- 当前版本：`1.2.5`
- 主要入口：
  - 侧边栏：`sidebar/sidebar.html`
  - 设置页：`options/options.html`
  - 对话管理页：`conversations/conversations.html`

## 2. 产品定位

Think Bot 是一个面向深度阅读与网页认知处理的浏览器扩展。它把网页提取、模板化提问、多模型对话、分支分析和历史管理串成一个连续工作流，更接近“认知工作台”，而不是单点式网页总结插件。

当前已实现的核心价值：

- 在普通网页中打开侧边栏并自动提取页面正文
- 基于当前页面内容向已配置模型发起对话
- 用快捷输入把常见分析动作做成可复用入口
- 对同一轮问题并发拉起多个分支模型回答
- 在独立页面中管理历史页面、历史会话和继续追问
- 支持导出会话、导入导出配置、云端同步配置与数据

## 3. 目标用户与使用前提

### 3.1 目标用户

- 高频阅读长文、报告、博客、新闻、论文解读的人
- 需要反复套用固定分析框架处理网页内容的人
- 需要比较多个模型回答质量的人
- 希望把网页阅读与 AI 对话沉淀为长期工作流的人

### 3.2 使用前提

- 用户需要自行配置 LLM API Key
- 扩展主要运行在普通网页中
- Chrome 内部页、扩展管理页等受限页面不能直接使用侧边栏
- 当前内置语言为中文和英文

## 4. 整体信息架构

当前产品由三个主要用户界面组成。

### 4.1 侧边栏

用于处理“当前网页”的即时阅读、提取、提问和分支分析。

### 4.2 设置页

用于管理模型、快捷输入、主题语言、内容提取参数、同步、黑名单和配置导入导出。

### 4.3 对话管理页

用于按网页维度查看历史页面、搜索、恢复会话、继续对话、编辑标题和删除页面数据。

## 5. 当前视觉与交互风格

当前界面更偏工具型工作台，而不是卡片式仪表盘。

- 主色为蓝色，用于当前选中、主按钮和结构强调
- 辅助强调色为橙色，主要用于当前激活状态的底部强调线
- 大面积背景为白色或浅灰色，优先服务长文本阅读
- 整体更强调信息密度和可操作性，而不是装饰感

共同视觉特征：

- 顶部工具按钮多为方形描边按钮
- 快捷输入呈现为高密度、多行换行的标签矩阵
- 正文区和回答区更像文档阅读区，不是气泡式 IM
- 分支结果以完整阅读列呈现，而不是摘要小卡片

## 6. 核心产品对象

### 6.1 页面 Page

一个网页 URL 对应一份页面缓存对象，包含页面元数据、提取内容和页面级状态。

### 6.2 会话 Conversation

一个页面下可以有多组会话内容。默认聊天与快捷输入标签聊天分开组织。

### 6.3 标签页 Tab

这里的 Tab 指侧边栏内部的会话标签，而不是浏览器 Tab。

- 默认存在 `Chat`
- 每条快捷输入对应一个额外标签
- 每个标签有独立聊天记录、加载状态和内容状态

### 6.4 快捷输入 Quick Input

快捷输入是一条可复用的提示模板，包含：

- 唯一 ID
- 显示名称
- 发送文本模板
- 自动触发开关
- 专属分支模型配置

### 6.5 分支 Branch

一次回答可以由多个模型并行生成多个分支结果。每个分支包含模型身份、状态和内容。

### 6.6 模型 Model

模型配置支持多条并存，包含显示名称、Provider、连接参数、模型标识和启用状态。

## 7. 入口行为

### 7.1 点击扩展图标

当前实现区分两类页面。

- 普通网页：
  - 打开 Side Panel
  - 向侧边栏发送 `SIDEBAR_OPENED`
  - 侧边栏执行页面数据加载与黑名单检查
- 受限页面：
  - 不进入正常侧边栏工作流
  - 直接打开对话管理页
  - 必要时给出受限页面提示

### 7.2 扩展图标右键菜单

已实现一个上下文菜单入口，用于直接打开对话管理页。

### 7.3 首次安装

首次安装时，会根据浏览器语言打开对应的快速上手文档。

## 8. 侧边栏功能定义

### 8.1 页面目标

侧边栏负责“当前网页即时处理”的主流程：

- 抽取网页正文
- 展示提取内容
- 快速发起 AI 提问
- 执行快捷输入分析动作
- 对同一轮问题进行分支式多模型分析

### 8.2 界面结构

#### 8.2.1 顶部控制区

左侧是提取相关操作，右侧是页面级和系统级操作。

已实现控件：

- `Jina` 提取按钮
- `Readability` 提取按钮
- 复制提取内容
- 重新提取
- 清空当前页面数据
- 打开对话管理页
- 打开设置页
- 打开 GitHub 项目页

#### 8.2.2 内容提取区

位于顶部工具区和快捷输入区之间，用于展示当前网页抽取结果。

已实现状态：

- 正常显示
- 加载中
- 提取失败

已实现交互：

- 复制提取内容
- 切换提取方式
- 重新提取
- 垂直拖拽调整高度

#### 8.2.3 快捷输入标签区

位于提取区和聊天区之间，是当前产品辨识度最高的区域之一。

当前行为：

- 默认存在 `Chat`
- 每条快捷输入生成一个额外标签
- 标签可显示 `loading` 和 `has-content`
- 点击标签可切换对应会话

当前产品心智：

- 它不是简单 Tab 条
- 它更像分析工作流总面板
- 同时承担导航与状态指示器角色

#### 8.2.4 聊天区

用于展示用户消息、主回答和分支回答。

当前支持：

- 普通用户消息
- 普通助手消息
- 分支式助手消息
- 加载中的分支列
- 错误分支

#### 8.2.5 图片预览区

用户粘贴图片后，会在输入区上方显示图片预览，并支持移除。

#### 8.2.6 输入区

包含：

- 多行文本输入框
- 模型选择下拉框
- 发送按钮
- “包含页面内容”开关
- 清空/停止按钮
- 导出按钮

输入区支持高度拖拽，按钮布局会随输入区高度变化。

### 8.3 侧边栏核心能力

#### 8.3.1 页面内容提取

当前支持两种提取方式：

- `Readability`
  - 默认提取方式
  - 本地基于页面 HTML 做正文抽取
- `Jina`
  - 通过 Jina 接口提取
  - 支持可选 API Key
  - 支持自定义响应模板

行为规则：

- 打开侧边栏时自动加载页面信息
- 页面已有缓存时优先展示缓存
- content script 未连接时，系统会尝试自动刷新页面后重试
- 部分重试流程中，Readability 失败后会回退到 Jina

#### 8.3.2 问答与对话

当前支持：

- 纯文本提问
- 粘贴图片并随消息发送
- 选定当前模型发送
- 将页面正文带入上下文
- 流式接收回答
- 中途取消请求

发送规则：

- 点击发送按钮发送
- `Enter` 发送
- `Shift + Enter` 换行

#### 8.3.3 “包含页面内容”开关

这是侧边栏的重要上下文控制开关。

- 打开时：发送消息会附加页面提取内容
- 关闭时：仅保留用户文本和 system prompt
- 该状态按页面保存并恢复

#### 8.3.4 快捷输入

快捷输入是最重要的效率功能之一。

当前行为：

- 每个快捷输入在侧边栏中表现为一个标签
- 手动切换标签时，只切换视图，不自动发送
- 触发快捷输入动作时，会把模板内容作为消息发送
- 每个快捷输入会话独立保存，不与默认 `Chat` 混存

默认内置了多条中文分析模板，例如：

- 概括
- 缩写
- 重构
- 反直觉
- 审计
- 第一性原理
- 意图分析

#### 8.3.5 自动触发快捷输入

快捷输入支持 `autoTrigger`。

行为规则：

- 页面内容加载完成后，检查所有开启自动触发的快捷输入
- 自动触发在后台执行，不强制切换当前可见标签
- 页面内容为空时不触发
- 该标签已有历史或已初始化时不重复触发
- 自动触发时可强制带入页面内容

#### 8.3.6 分支模型并发分析

支持“主模型回答 + 多个分支模型并发回答”的结构。

分支模型来源：

- 全局分支模型
- 快捷输入专属分支模型

行为规则：

- 触发快捷输入时，会合并全局和专属分支模型
- 为每个模型生成独立分支列
- 每个分支有独立的 `loading / done / error`
- 分支数量较多时，UI 会增强横向浏览能力

#### 8.3.7 单条消息分支与重试

当前不只是自动分支，也支持在已有回答上继续扩展。

已实现能力：

- 用户消息支持重试
- 重试以新的 branch 发起，不覆盖原回答
- 分支支持继续新增
- 分支支持删除
- 加载中的分支支持停止并删除

#### 8.3.8 分支内容预览

每个分支可在独立 Overlay 中预览完整内容。

支持：

- Markdown 渲染
- 点击遮罩关闭
- `Esc` 关闭
- 拖拽调整预览面板尺寸

#### 8.3.9 消息操作

用户消息支持：

- 编辑
- 重试
- 复制纯文本
- 复制 Markdown

助手消息支持：

- 定位到该消息顶部
- 定位到该消息底部
- 复制纯文本
- 复制 Markdown

#### 8.3.10 清空当前会话

清空按钮具备双重语义：

- 若存在进行中的请求，先尝试取消
- 再清空当前标签对应的聊天记录

同时会清理 loading 状态、更新标签内容状态，并重置自动触发相关初始化状态。

#### 8.3.11 导出会话

当前导出格式为 Markdown。

导出内容包含：

- 页面标题
- 页面 URL
- system prompt
- 用户消息
- 助手消息
- 分支消息与模型信息

### 8.4 侧边栏异常与边界

#### 8.4.1 黑名单拦截

当 URL 命中黑名单时，侧边栏不会直接工作，而是先弹出确认层。

用户可选：

- 继续使用
- 取消并关闭侧边栏

默认黑名单包含搜索页，例如 Google、Bing、百度搜索。

#### 8.4.2 受限页面处理

Chrome 内部页等受限环境下，不进行内容提取与对话。

#### 8.4.3 流式恢复

重新打开侧边栏时，如果发现某个标签仍有 loading state，会尝试恢复加载中的 UI。

## 9. 设置页功能定义

### 9.1 页面目标

设置页负责管理所有长期配置，包括模型、快捷输入、提取行为、同步、黑名单和导入导出。

### 9.2 页面结构

采用左侧纵向导航、右侧详情区的后台式结构。

一级栏目：

- Basic Settings
- Quick Input Tabs
- Language Models
- Cloud Sync
- Blacklist Settings

顶部常驻全局操作：

- 导出配置
- 导入配置
- 恢复默认
- 保存
- 保存并同步

另外存在错误显示区，用于展示最近一次捕获的错误。

### 9.3 Basic Settings

包含：

- 本地缓存占用查看与清理
- 默认模型选择
- 全局分支模型配置
- 主题配置
- 语言配置
- 自定义 System Prompt
- Filter COT
- 内容提取设置

内容提取设置包括：

- 默认提取内容区域高度
- 默认提取方式
- Jina API Key
- Jina 响应模板

### 9.4 Quick Input Tabs

当前支持：

- 新增快捷输入
- 拖拽排序
- 折叠展开
- 名称与消息预览
- 自动触发开关
- 软删除
- 专属 branch model 配置

同时支持从云端导入快捷输入模板，当前默认来源是项目公开的 `quick_input_tabs.json`。

### 9.5 Language Models

当前支持的 Provider：

- OpenAI Compatible
- Google Gemini
- Azure OpenAI
- Anthropic Claude

通用能力：

- 显示名称
- 启用/停用
- 复制模型
- 删除模型
- 拖拽排序

Provider 专属配置能力覆盖了 Base URL、API Key、Model、Deployment、Token、Temperature、Tools、Thinking Budget 等差异字段。

### 9.6 Cloud Sync

同步能力用于把配置和数据与云端存储对齐。

当前支持的存储后端：

- Gist
- WebDAV

当前支持：

- 连接测试
- 保存并同步
- 同步状态展示
- 最近同步时间展示

### 9.7 Blacklist Settings

黑名单能力用于阻止用户在不适页面上误触发工作流。

当前支持：

- 黑名单规则列表
- 新增、编辑、删除规则
- 默认规则恢复
- 黑名单匹配测试

### 9.8 配置导入导出

当前支持：

- 导出完整配置
- 导入配置文件并恢复本地配置

### 9.9 错误信息展示

设置页会读取最近一次错误信息并展示，便于排查侧边栏、对话管理页或同步流程异常。

## 10. 对话管理页功能定义

### 10.1 页面目标

对话管理页不是归档页，而是历史会话的浏览与继续工作入口。

### 10.2 页面结构

采用左右双栏布局，并带横向拖拽分隔条。

左栏包含：

- 搜索框
- 历史页面列表
- 空状态视图

单条页面项展示：

- 网站图标
- 外链打开图标
- 页面标题
- 删除按钮

右栏包含：

- 页面标题
- 页面 URL
- 复制提取内容按钮
- 提取内容区
- 快捷输入标签区
- 聊天区
- 图片预览区
- 输入区

### 10.3 页面列表功能

当前支持：

- 按最近更新时间加载历史页面
- 按标题或 URL 搜索
- 在新标签页打开原网页
- 删除页面数据

删除规则：

- 开启同步时使用软删除语义
- 未开启同步时使用硬删除语义

### 10.4 会话恢复与继续对话

选中历史页面后，会恢复：

- 页面标题与 URL

### 10.5 标题内联编辑

交互规则：

### 10.6 与侧边栏共享能力

对话管理页复用了大量侧边栏能力：

- 模型选择
- 快捷输入标签
- 分支消息
- 图片输入
- 导出
- 停止/清空
- Markdown 渲染

## 11. 产品交互规则

本章只保留用户视角下最重要的行为规则。

### 11.1 侧边栏交互规则

- 点击扩展图标后，若页面可用，则进入侧边栏工作流
- 页面命中黑名单时，优先弹确认层，而不是直接执行抽取
- 切换提取方式后，按钮状态必须同步切换
- 点击已激活的提取方式按钮时，不强制重新提取
- 点击“重新提取”时，必须沿用当前提取方式
- `Chat` 标签始终存在
- 快捷输入标签按配置顺序渲染
- 自动触发快捷输入不应打断用户当前正在查看的标签
- 发送时只有“文本非空”或“存在图片”才允许提交
- 发送时如果开启“包含页面内容”，必须附加当前页面内容
- 分支操作必须局部生效，不能误影响其他分支
- 清空时必须先尝试取消进行中的请求
- 导出为空时不应产出空文件

### 11.2 设置页交互规则

- 所有配置项改动后，页面进入“有未保存更改”状态
- 切换左侧导航不应丢失未保存表单
- 主题和语言允许即时预览，但保存后才持久化
- 默认模型下拉只展示启用且配置完整的模型
- 删除模型和快捷输入采用软删除语义
- 导入前必须做结构校验
- 导入成功后需要重建模型、快捷输入和相关控件状态
- 同步成功后应更新最近同步时间
- 同步失败后应保留错误信息供排查

### 11.3 对话管理页交互规则

- 点击左侧页面项后，右侧必须切换到对应页面上下文
- 点击“打开原网页”只开新标签，不改变右侧当前选中内容
- 点击删除按钮只删除目标页面，不应顺带触发选中
- 搜索输入时应实时过滤
- 搜索无结果时应展示空状态
- 标题保存失败时应回退旧标题，避免 UI 与数据不一致
- 继续对话区域的行为应尽量与侧边栏保持一致

## 12. 异常流

### 12.1 侧边栏异常流

- content script 未连接：
  - 识别为连接类错误
  - 自动刷新当前页面后重试
  - 若仍失败，展示失败提示和手动重试入口
- 页面内容提取失败：
  - 展示失败提示
  - 保留当前页面上下文
  - 允许切换提取方式或重新提取
- 流式请求异常终止：
  - 分支或主回答标记为 error
  - 清理 loading state
  - 恢复发送按钮可用
- 用户主动取消：
  - 视为用户操作而非系统错误
  - 结束 loading
  - 不弹误导性的失败提示
- 黑名单命中：
  - 阻断自动工作流
  - 展示确认层

### 12.2 设置页异常流

- 保存失败：
  - 展示具体错误
  - 不清空当前表单
- 同步失败：
  - 保留本地配置
  - 展示失败状态与错误信息
- 导入失败：
  - 不覆盖现有配置
  - 保持当前页面状态

### 12.3 对话管理页异常流

- 页面数据不存在或损坏：
  - 右侧显示空/错误状态
  - 左侧仍可继续切换其他页面
- 删除失败：
  - 保留当前选中项
  - 不提前清空右侧 UI
- 标题保存失败：
  - 恢复旧标题
  - 不保留半编辑态

## 13. 非功能要求

- 性能：
  - 侧边栏优先展示缓存
  - 列表和标签支持高密度展示，避免明显卡顿
  - 标签切换尽量避免整页重绘
- 稳定性：
  - 刷新或侧边栏重开后尽量恢复关键状态
  - 流式异常时必须回收 loading state
- 一致性：
  - 侧边栏与对话管理页的输入、标签、导出和分支交互尽量一致
  - “启用且配置完整”的模型定义在所有入口保持一致
- 可读性：
  - 长文本回答优先服务阅读
  - 分支模型名称必须清晰可见
  - 当前页面、URL、标签上下文必须清晰
- 安全与隐私：
  - API Key 仅保存在本地配置和用户显式配置的同步目标中
  - 黑名单能力应降低误触发风险

## 14. 当前边界与限制

- 用户必须自行配置模型和 API Key
- 侧边栏不能在 Chrome 受限页面中直接工作
- 当前仅支持 `Readability` 与 `Jina` 两种内容提取方式
- 当前仅支持 4 类模型 Provider
- 当前云同步仅支持 Gist 与 WebDAV
- 当前会话导出只提供 Markdown
- 当前黑名单以正则配置为主，更偏高级用户
- 当前历史会话主要按“页面 URL + 内部标签”组织，尚无项目或文件夹层级

## 15. 一句话总结

当前实现的 Think Bot 已经形成“网页提取 + 模板化提问 + 多模型分支 + 历史工作台 + 配置同步”的完整产品闭环。
