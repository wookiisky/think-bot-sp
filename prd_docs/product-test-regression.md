# Think Bot 测试与回归清单

## 1. 文档范围

本文档用于把“整体技术重构不能丢功能”落到可执行测试上，覆盖最小验收、分层回归和核心测试矩阵。

## 2. 使用原则

- 先覆盖高风险、高频、强耦合路径
- 优先保护用户可感知主流程
- 优先保护状态恢复、分支流式、配置持久化、同步和导入导出
- 每次较大重构至少跑完 `P0` 用例
- 改动高风险模块时，必须补对应自动化或半自动化回归

## 3. 测试分层

### 3.1 P0 核心保护

必须在每次较大重构后验证：

- 侧边栏打开与页面抽取
- 直接提问
- 快捷输入发送
- 自动触发快捷输入
- 分支流式完成、失败、停止
- 标签切换与历史恢复
- 对话管理页恢复历史页面
- 设置页保存模型与快捷输入
- 配置导入导出

### 3.2 P1 重要补充

建议在阶段性重构完成后验证：

- 主题与语言切换
- 页面标题编辑
- 黑名单拦截
- 清空页面数据
- 同步连接测试与同步状态展示
- 软删除与硬删除分支

### 3.3 P2 扩展保护

适合逐步自动化：

- 压缩与解压兼容
- 异常错误展示细节
- 流式恢复
- 复杂 Markdown 渲染
- 大量历史页面列表性能

## 4. 最小回归验收清单

### 4.1 侧边栏

- 可从普通网页打开
- 可在受限页面退化到对话管理页
- 首次打开能抽取页面内容
- 提取方式切换正常
- 输入提问正常
- 图片发送正常
- 快捷输入仍可触发
- 自动触发仍可后台运行
- 分支仍能出现并显示模型名
- 分支预览仍可打开
- 清空仍能取消并清理状态
- 导出仍能产出正确 Markdown

### 4.2 对话管理页

- 左侧页面列表仍能加载
- 搜索仍有效
- 选中页面后右侧内容能恢复
- 标题编辑仍可保存
- 删除页面仍正确处理软删或硬删
- 右侧继续对话仍可用

### 4.3 设置页

- 模型新增、复制、删除、启停、排序正常
- 默认模型候选正确
- 快捷输入新增、删除、排序、自动触发配置正常
- branch model 多选正常
- 主题与语言切换正常
- 导入导出正常
- Gist 或 WebDAV 测试正常
- 保存、保存并同步、手动同步状态正常

## 5. 建议测试形态

### 5.1 单元测试优先覆盖

- 纯函数状态派生
- URL 归一化
- 配置构建与兼容解析
- 聊天记录和页面数据 key 生成
- 软删除语义
- 导出 Markdown 结构拼接

### 5.2 集成测试优先覆盖

- background handler 与 storage 交互
- sidebar 与 conversations 页面响应消息事件
- 快捷输入与 tab state 联动
- loading state 的保存、读取与恢复

### 5.3 端到端测试优先覆盖

- 用户从网页打开侧边栏并完成一轮分析
- 用户在对话管理页恢复并继续历史对话
- 用户在设置页修改配置后返回使用

## 8. 核心测试矩阵

### 8.1 侧边栏入口与内容提取

#### TB-001 打开侧边栏并自动抽取页面

- 优先级：P0
- 前置条件：
  - 当前位于普通网页
  - 扩展已安装
- 步骤：
  1. 点击扩展图标
  2. 等待侧边栏打开
  3. 观察提取内容区
- 预期：
  - 侧边栏成功打开
  - 页面内容开始加载或直接显示缓存
  - 顶部提取按钮与内容区状态一致

#### TB-002 在受限页面点击扩展图标

- 优先级：P1
- 前置条件：
  - 当前位于 Chrome 内部页或类似受限页面
- 步骤：
  1. 点击扩展图标
- 预期：
  - 不进入正常侧边栏工作流
  - 打开对话管理页或出现受限页面提示

#### TB-003 点击扩展图标右键菜单打开历史页

- 优先级：P1
- 前置条件：
  - 扩展已安装
- 步骤：
  1. 打开扩展图标右键菜单
  2. 点击 `Conversations`
- 预期：
  - 直接打开对话管理页
  - 不依赖当前页面是否可注入

#### TB-004 首次安装打开快速上手文档

- 优先级：P1
- 前置条件：
  - 在干净浏览器环境首次安装扩展
- 步骤：
  1. 安装扩展
  2. 观察首次打开的新标签页
- 预期：
  - 自动打开快速上手文档
  - 文档语言与浏览器语言一致

#### TB-005 切换提取方式

- 优先级：P0
- 前置条件：
  - 侧边栏已打开
  - 页面有可提取内容
- 步骤：
  1. 记录当前提取方式
  2. 点击另一种提取方式按钮
  3. 等待结果刷新
- 预期：
  - 当前提取方式高亮切换
  - 提取结果刷新
  - 状态写回当前页面状态

#### TB-006 重新提取当前页面

- 优先级：P1
- 前置条件：
  - 当前页面已完成一次提取
- 步骤：
  1. 点击重试提取按钮
- 预期：
  - 进入加载态
  - 使用当前提取方式重新执行
  - 成功后结果刷新

#### TB-007 顶部控制区按钮行为

- 优先级：P1
- 前置条件：
  - 侧边栏已打开
  - 当前页面已有提取内容
- 步骤：
  1. 点击复制提取内容
  2. 点击打开对话管理页
  3. 点击打开设置页
  4. 点击打开 GitHub 项目页
- 预期：
  - 提取内容可被复制
  - 历史页、设置页、GitHub 分别在正确页面打开
  - 不影响当前标签聊天状态

#### TB-008 提取区高度拖拽

- 优先级：P1
- 前置条件：
  - 侧边栏已打开
  - 提取区已显示内容
- 步骤：
  1. 拖拽提取区分隔条
  2. 观察提取区与聊天区布局
- 预期：
  - 提取区高度随拖拽变化
  - 布局稳定，不遮挡核心操作区

### 8.2 消息发送与直接对话

#### TB-010 发送一条普通文本消息

- 优先级：P0
- 前置条件：
  - 至少有一个可用模型
  - 侧边栏已打开
- 步骤：
  1. 输入一条测试问题
  2. 点击发送
- 预期：
  - 用户消息立即写入聊天区
  - 当前 tab 进入 loading
  - 模型开始流式返回
  - 完成后 loading 结束，回答保存

#### TB-011 使用 Enter 发送，Shift+Enter 换行

- 优先级：P1
- 前置条件：
  - 输入框可编辑
- 步骤：
  1. 输入文本，按 `Shift+Enter`
  2. 确认未发送且出现换行
  3. 按 `Enter`
- 预期：
  - `Shift+Enter` 只换行
  - `Enter` 发送消息

#### TB-012 切换“包含页面内容”开关

- 优先级：P0
- 前置条件：
  - 当前页面存在提取内容
- 步骤：
  1. 记录当前开关状态
  2. 点击开关
  3. 刷新或切页后返回当前页面
- 预期：
  - UI 状态即时切换
  - 页面级状态被保存
  - 返回该页面后开关状态按页面恢复

#### TB-013 图片预览添加与移除

- 优先级：P1
- 前置条件：
  - 输入区可编辑
- 步骤：
  1. 粘贴一张图片
  2. 观察输入区上方预览
  3. 移除该图片
- 预期：
  - 图片预览正确显示
  - 移除后预览消失
  - 移除后的发送不会再带上该图片

#### TB-014 输入区高度拖拽

- 优先级：P1
- 前置条件：
  - 侧边栏已打开
- 步骤：
  1. 拖拽输入区高度
  2. 观察按钮布局变化
- 预期：
  - 输入区高度可调整
  - 按钮布局随高度变化
  - 发送、停止、导出仍可正常点击

### 8.3 快捷输入与标签

#### TB-020 快捷输入标签切换

- 优先级：P0
- 前置条件：
  - 至少存在 2 条快捷输入
- 步骤：
  1. 依次点击 `Chat`、快捷输入 A、快捷输入 B
- 预期：
  - 当前激活标签正确切换
  - 每个标签展示自己的聊天记录
  - 标签状态样式同步更新

#### TB-021 手动触发快捷输入

- 优先级：P0
- 前置条件：
  - 存在一条有效快捷输入
- 步骤：
  1. 触发该快捷输入发送
- 预期：
  - 用户侧消息写入当前标签
  - assistant loading UI 正确生成
  - 完成后聊天记录保存到 `url#tabId`

#### TB-022 自动触发快捷输入

- 优先级：P0
- 前置条件：
  - 至少有一条 `autoTrigger=true` 的快捷输入
  - 页面有可用提取内容
- 步骤：
  1. 打开侧边栏
  2. 等待页面数据加载完成
- 预期：
  - 自动触发快捷输入在后台执行
  - 不强制切换当前可视标签
  - 对应标签进入 loading，再进入 has-content

#### TB-023 自动触发防重复

- 优先级：P0
- 前置条件：
  - 某 autoTrigger 标签已有历史记录
- 步骤：
  1. 关闭并重新打开侧边栏
- 预期：
  - 不重复触发该 autoTrigger 标签

### 8.4 分支与流式

#### TB-030 快捷输入触发多分支回答

- 优先级：P0
- 前置条件：
  - 存在全局或快捷输入专属 branch model
- 步骤：
  1. 触发该快捷输入
- 预期：
  - 主回答与分支回答按规则生成
  - 每个分支显示模型名
  - 分支可独立完成

#### TB-031 分支预览

- 优先级：P0
- 前置条件：
  - 至少有一个完成态分支
- 步骤：
  1. 点击分支预览按钮
  2. 观察 overlay
  3. 使用遮罩或 `Esc` 关闭
- 预期：
  - overlay 正确显示完整内容
  - 内容按 Markdown 渲染
  - 可正常关闭

#### TB-032 分支停止并删除

- 优先级：P0
- 前置条件：
  - 至少有一个 loading 分支
- 步骤：
  1. 点击该分支停止或删除按钮
- 预期：
  - 仅目标分支被停止
  - 目标分支 UI 删除或结束
  - 其他分支不受影响

#### TB-033 分支错误回收

- 优先级：P0
- 前置条件：
  - 构造一个必然失败的模型请求
- 步骤：
  1. 发起请求
  2. 等待错误返回
- 预期：
  - 分支进入 error 展示
  - loading state 被清理
  - 发送按钮恢复可用

#### TB-034 侧边栏重开后恢复 loading UI

- 优先级：P1
- 前置条件：
  - 某 tab 或 branch 正在 loading
- 步骤：
  1. 关闭侧边栏
  2. 重新打开侧边栏
- 预期：
  - 当前 loading state 被正确恢复
  - 完成或错误后状态可继续正确回收

### 8.8 同步与删除语义

#### TB-070 Gist 连接测试

- 优先级：P1
- 自动化口径：env-gated，仅在发布前通过 `npm run test:release:phase6` 执行真实 provider smoke
- 前置条件：
  - 已填写有效 Gist Token 与 Gist ID
- 步骤：
  1. 执行连接测试或保存并同步
- 预期：
  - 返回成功状态
  - 状态区展示最近同步时间或成功信息

#### TB-071 WebDAV 连接测试

- 优先级：P1
- 自动化口径：env-gated，仅在发布前通过 `npm run test:release:phase6` 执行真实 provider smoke
- 前置条件：
  - 已填写有效 WebDAV 配置
