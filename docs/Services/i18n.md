# 国际化服务

## 1. 模块定位

国际化服务负责加载中英文轻量语言资源，提供 `t(key)` 查询和语言切换能力。

## 2. 核心抽象

- `LocaleCode`
- `LocaleResource`
- `TranslationKey`
- `TranslationParams`

## 3. 能力边界

负责：

- 解析 `locales/*.yml`
- 生成 typed key
- 提供运行时翻译函数
- 管理当前语言和回退链

不负责：

- 管理复杂 ICU 规则。
- 处理三种以上语言。
- 把资源直接暴露给业务页面自行解析。

## 4. 对外接口

- `loadResources()`
- `setLocale(locale)`
- `getLocale()`
- `t(key, params?)`

文件约束：

- 只允许平铺 `key: 文本`
- 语言仅 `zh-CN` 与 `en`

## 5. 关键流程

- 应用启动时加载全部语言资源。
- 从配置读取当前语言。
- UI 使用 `t(key)` 渲染文本。
- 设置页切换语言后即时刷新当前页面并持久化到配置。
- 工作台类页面需要把翻译入口下沉为可复用函数，避免 `SidebarShell`、`ConversationsShell`、`ChatInput`、`ChatThread` 各自硬编码同义文案。
- icon button、状态 icon、编辑态输入框等无常驻可见文字的控件，必须通过 i18n 提供 `aria-label` 或 `title`。

## 6. 错误与异常处理

- 资源解析失败：
  - 回退到 `zh-CN`
- key 缺失：
  - 当前语言找不到时回退到 `zh-CN`
  - `zh-CN` 也缺失时显示 key

## 7. 数据与状态

- 静态资源：
  - `locales/zh-CN.yml`
  - `locales/en.yml`
- 持久化：
  - `ExtensionConfig.basic.language`

## 8. 依赖与协作模块

- `dao/locale-repository.md`
- `dao/config-repository.md`
- `Workspace/settings.md`

## 9. 约束与禁止事项

- 不使用重量级国际化框架。
- 不允许在语言文件中写嵌套结构。
- 不允许业务组件拼接未受控 key。
- 不允许同一交互语义在不同工作台维护两套 key，例如“编辑”“重试”“导出”“停止”“恢复中”必须走共享 key。

## 10. 测试要求

- 职责测试：加载资源、切换语言、回退。
- 边界测试：缺失 key、空文本。
- 错误流测试：语言文件解析失败。
- 不变量测试：两种语言 key 集合一致。
