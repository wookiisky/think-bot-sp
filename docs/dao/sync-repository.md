# 同步仓储

## 1. 关联模块

- 同步服务
- 设置页
- 删除语义

## 2. 访问数据域

- `DataSchema/sync-snapshot.md`
- `DataSchema/config.md`
- `DataSchema/page.md`
- `DataSchema/conversation.md`

## 3. 提供的能力

- 聚合本地全量数据为快照。
- 维护本地同步状态和最近同步时间。
- 写入和读取墓碑删除记录。
- 提供软删除过滤视图。
- 提供本地快照与远端快照的对象级合并视图。

## 4. 事务边界与并发约束

- 快照导出必须基于同一时间点的逻辑视图。
- 删除写墓碑与本地过滤规则必须保持一致。
- 同步失败不回滚用户本地有效数据。
- 墓碑过滤必须先于页面和会话合并执行。

## 5. 上层依赖边界

- 上层只关心快照和同步状态。
- 不直接依赖底层页面或会话 key。
- 不向上层暴露旧同步结构兼容逻辑。

## 6. 需要验证的点

- 软删除与硬删除分支正确。
- 快照结构稳定。
- 同步失败后的本地状态正确。
- 墓碑过滤正确阻止已删数据复活。
- 配置、页面、会话对象级合并正确。
