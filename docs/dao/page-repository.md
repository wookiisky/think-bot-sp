# 页面仓储

## 1. 关联模块

- 侧边栏
- 对话管理页
- 提取服务
- 导出

## 2. 访问数据域

- `DataSchema/page.md`

## 3. 提供的能力

- 根据 URL 归一化规则读写页面记录。
- 获取单页面缓存。
- 批量获取历史页面元数据。
- 按标题或 URL 搜索页面。
- 更新标题、图标、提取内容、提取方式、页面级状态。
- 更新页面级 `includePageContent`，且不覆盖正文与 `promptTabStates`。
- 更新页面下 `promptTab` 初始化状态和自动触发状态。
- 为 side panel bootstrap 提供页面缓存、页面级状态和 `promptTab` 去重状态读取。
- 提供本地页面缓存统计与可安全清理的数据访问能力。
- 统计口径只覆盖可回收的本地页面缓存，不包括配置和同步状态。
- 级联删除页面相关数据。

## 4. 事务边界与并发约束

- 页面记录写入与更新时间更新必须一起完成。
- 页面硬删除必须与会话和 loading 清理保持同一操作序列。
- 标题编辑失败时不更新列表展示。
- 缓存统计与缓存清理必须只作用于可回收页面数据，不能误删配置或同步状态。
- 清理动作只清理安全可回收的数据，不能影响配置仓储和同步仓储的持久化内容。

## 5. 上层依赖边界

- 上层只关心 `PageRecord` 和查询结果。
- 不暴露具体 key 拼接规则。
- URL 归一化只能由仓储内部统一实现。

## 6. 需要验证的点

- URL 归一化一致性。
- 列表排序正确。
- 搜索正确。
- 删除页面级联清理正确。
- 提取失败不覆盖旧缓存。
- `promptTab` 初始化状态与清空重置语义正确。
- side panel bootstrap 读取到的页面状态与持久化事实一致。
- 本地缓存统计与清理结果正确。
