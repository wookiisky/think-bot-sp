# 语言资源数据域

## 1. 数据域概述

- 名称：`LocaleResource`
- 存储类型：扩展打包静态文件
- 业务意义：提供中文和英文界面文案。
- 所属模块：设置页、侧边栏、对话管理页、国际化服务。
- 上下游依赖：
  - 上游：静态语言资源文件。
  - 下游：运行时 `t(key)` 查询。

## 2. 字段定义

文件形态：

- `locales/zh-CN.yml`
- `locales/en.yml`

文件内字段：

- `locale`
  - 类型：文件名隐含
  - 取值：`zh-CN`、`en`
- `key`
  - 类型：`string`
  - 必填：是
  - 含义：平铺命名键。
- `text`
  - 类型：`string`
  - 必填：是
  - 含义：对应显示文本。

## 3. 索引与限制

- 仅允许单层 `key: 文本`。
- 不允许嵌套对象。
- 不允许 ICU、富格式 DSL 或复杂复数规则。
- 默认语言为 `zh-CN`，回退链为：当前语言 -> `zh-CN` -> key 原文。

## 4. 读写路径

- 谁读：
  - i18n service
  - 三个 UI 页面
- 谁写：
  - 仅源码维护者在代码仓库中修改
- 典型查询：
  - 按 key 取值
  - 生成 typed key 列表

## 5. 生命周期与风险

- 随构建产物发布。
- 设置页切换语言只改变当前活跃语言，不修改资源文件。
- 风险：
  - 缺失 key 导致界面显示 key。
  - 两种语言 key 不一致导致类型或运行时回退。

## 6. 测试要求

- 两种语言 key 对齐测试。
- 缺失 key 回退测试。
- 语言切换即时刷新测试。
- typed key 生成测试。
