# 黑名单规则数据域

## 1. 数据域概述

- 名称：`BlacklistRule`
- 存储类型：`chrome.storage.local`
- 业务意义：定义哪些 URL 会阻断侧边栏自动工作流。
- 所属模块：设置页、侧边栏入口、黑名单服务。
- 上下游依赖：
  - 上游：设置页编辑。
  - 下游：侧边栏确认层。

## 2. 字段定义

- `id`
  - 类型：`string`
  - 必填：是
- `pattern`
  - 类型：`string`
  - 必填：是
  - 含义：正则或约定格式规则。
- `description`
  - 类型：`string`
  - 必填：否
- `enabled`
  - 类型：`boolean`
  - 必填：是
  - 默认值：`true`
- `isDefault`
  - 类型：`boolean`
  - 必填：是

## 3. 索引与限制

- 主来源仍归属配置主对象。
- `id` 唯一。
- 默认规则可恢复，但不允许因恢复操作丢失用户自定义规则。

## 4. 读写路径

- 谁读：
  - background 黑名单服务
  - 设置页
- 谁写：
  - 仅 background 配置写入层
- 典型查询：
  - 初始化时检查当前 URL。
  - 设置页测试规则。

## 5. 生命周期与风险

- 随配置长期存在。
- 删除或禁用后立即影响后续页面打开行为。
- 风险：
  - 误匹配导致普通页面被拦截。
  - 正则错误导致设置页保存失败。

## 6. 测试要求

- 默认规则恢复测试。
- 命中与未命中测试。
- 非法规则校验测试。
- 启用/禁用即时生效测试。
