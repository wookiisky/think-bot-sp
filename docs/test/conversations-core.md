# 对话管理页核心回归

## 1. 关联能力

- 历史页面列表
- 搜索
- 恢复会话
- 左右栏拖拽
- 继续对话
- 标题编辑
- 模型选择、图片输入、停止、清空、导出
- 打开原网页与复制提取内容
- 删除页面

## 2. 核心职责清单

- 按页面维度展示历史。
- 恢复页面标题、URL、提取内容、标签和聊天。
- 维持左右双栏工作台布局和拖拽分隔条。
- 继续对话行为与侧边栏一致。
- 支持复制提取内容和打开原网页但不打断当前上下文。
- 支持标题编辑、页面删除和删除语义分流。
- 支持主分支切换、助手分支重试和响应式多列分支展示。
- 支持共享分支预览层，且关闭后不影响当前页面、标签和输入草稿。

## 3. 验证点

- 正常流：
  - 列表加载。
  - 搜索过滤。
  - 选中页面恢复完整上下文。
  - 右侧恢复链路先展示已持久化页面与会话，再按需恢复 loading，不依赖 background 主动推送首屏消息。
  - 左右栏拖拽后布局更新且当前上下文不丢失。
  - 继续提问。
  - 图片输入、模型切换、停止、清空、导出可用，且输入区首次打开按实际内容高度展示。
  - 打开原网页后右侧当前选中上下文保持不变。
  - 标题支持 `Enter` 保存、`Esc` 取消、失焦保存。
- 编辑用户消息、继续新增分支和 Markdown 展示行为与侧边栏一致。
- 快捷输入 tab 只保留底部彩线和 loading 边框，不再显示额外状态 icon。
- 继续新增分支先弹出模型列表，选中模型后只追加一个分支，并沿用侧边栏的同模型编号规则。
- 多分支消息的响应式并行多列行为与侧边栏一致。
- 分支预览层行为与侧边栏一致：打开入口、遮罩关闭、`Esc` 关闭和本地尺寸拖拽协议保持同源。
- 当前轮主分支切换、用户消息重试、助手分支重试语义与侧边栏一致。
  - 有效会话导出成功，空会话导出明确失败。
- 错误流：
  - 页面数据损坏。
  - 标题保存失败。
  - 删除失败。
- 异常流：
  - 页面有 loading 状态时恢复。
  - 分支 loading 与错误状态恢复。
  - provider 错误文本直出恢复。

## 4. 关键边界条件

- 空列表。
- 搜索无结果。
- 当前选中页面被删除。
- 同步开启时删除走软删。
- 同步关闭时删除走硬删。
- 空会话页面不导出空文件。

## 5. 历史高风险场景

- 列表来源错误地依赖聊天内容反推。
- 删除按钮触发页面选中。
- 标题编辑失败后 UI 未回滚。
- 对话页输入区与侧边栏交互漂移。
- 拖拽布局导致当前标签和草稿丢失。
- 右侧恢复顺序错误，先渲染空白再覆盖历史，造成闪烁或错误导出。

## 6. 测试数据与环境

- 多条历史页面记录
- 同一页面下多个标签会话
- 含 loading 的历史记录
- 含分支结果和分支错误的历史记录
- 同步开启与关闭两套删除场景
