# 同步与删除回归

## 1. 关联能力

- Gist 同步
- WebDAV 同步
- 配置快照推送

## 2. 核心职责清单

- 正确读取远端、合并并推送快照。
- 正确测试连接。
- 最近同步时间展示正确。
- 页面删除时同步开启会先写入 tombstone，再清理本地页面、会话与 loading。
- 本地回写后不会留下孤儿 page / conversation / loading。

## 3. 验证点

- 正常流：
  - Gist 连接测试成功。
  - WebDAV 连接测试成功。
  - 远端文件不存在时按首次同步处理。
  - 手动同步更新最近同步时间。
  - 同步反馈展示快照大小。
  - 同一页面的 Readability / Jina 缓存按各自更新时间合并。
- 错误流：
  - 鉴权失败。
  - 网络失败。
  - 远端不可读。
  - 远端格式非法。
- 异常流：
  - service worker 启动后再注入测试 provider。
  - 同步失败后再次重试。
  - 本地删远端改、远端删本地改。
  - 页面已删除时，即使某个方法缓存更新时间更晚也不能复活页面。

## 4. 关键边界条件

- 空数据集。
- 缺少 Gist Token / Gist ID。
- 缺少 WebDAV URL。

## 5. 历史高风险场景

- service worker 重启或惰性回收后，测试桩注入失效导致误走真实网络。
- 同步失败覆盖本地有效数据。
- 最近同步时间显示与真实状态不一致。
- tombstone 未优先生效导致已删页面复活。

## 6. 测试数据与环境

- 启用同步的配置
- 远端错误桩
