# 侧边栏核心回归

## 1. 关联能力

- 侧边栏打开
- 页面提取
- 文本/图片发送
- 快捷输入
- 消息编辑与分支操作
- 分支预览
- 清空与导出

## 2. 核心职责清单

- 在普通网页打开 side panel。
- 展示缓存或完成提取。
- 支持直接提问和快捷输入。
- 正确展示主回答和分支。
- 支持编辑用户消息、继续新增分支、停止/删除目标分支。
- 支持停止、清空、导出。

## 3. 验证点

- 正常流：
  - 打开侧边栏并显示内容。
  - 切换提取方式并刷新。
  - 文本发送、图片发送成功。
  - 自动触发快捷输入在后台完成。
  - 编辑用户消息后重发成功且旧结果被裁剪。
  - 继续新增分支只追加目标消息的分支。
- 错误流：
  - content script 未连接。
  - 提取失败。
  - 流式中断。
- 异常流：
  - side panel 关闭重开恢复 loading。
  - 用户取消请求。
  - content script 断连后自动刷新一次再重试。

## 4. 关键边界条件

- 黑名单命中。
- 受限页面退化。
- 空内容页面。
- 快捷输入已有历史时防重复触发。
- 清空当前标签后再次进入页面可重新自动触发。

## 5. 历史高风险场景

- 标签 loading 与 has-content 状态错乱。
- 分支停止误影响其他分支。
- 清空时未先取消进行中请求。
- 编辑用户消息后保留了过期回答。

## 6. 测试数据与环境

- 普通长文页
- 搜索结果页
- 受限页
- 至少 1 个完整可用模型
- 至少 1 条 autoTrigger 快捷输入
