# 侧边栏

## 1. 模块定位

侧边栏是“当前网页即时处理”的主工作台，负责页面提取、快速提问、快捷输入、分支分析和导出。

术语约定：

- `browserTab`：Chrome 浏览器标签页。
- `promptTab`：侧边栏内的 `Chat` 或某个快捷输入标签。

## 2. 界面布局映射

侧边栏界面按设计图拆成 4 个稳定区域：

- 顶部控制区：
  - 固定在顶部。
  - 左侧是提取相关控件：`Readability`、`Jina`、复制提取内容、重新提取。
  - 右侧是页面级和系统级控件：清空当前页面数据、打开历史页、打开设置页、打开 GitHub。
  - 控件视觉统一采用 icon button，动作语义通过 `aria-label`、`title` 和 i18n 资源提供，避免页面级动作与标签级动作混排成长文本按钮。
- 提取内容区：
  - 作为常驻独立区域显示，不因切换 `promptTab` 而隐藏。
  - 位于顶部控制区下方、`promptTab` 区上方，承担“当前页面正文阅读区”职责。
  - 支持正常、加载中、提取失败三种状态，支持复制与垂直拖拽调整高度。
- 快捷输入标签区：
  - 位于提取内容区下方。
  - 默认包含 `Chat` promptTab，其余 promptTab 来自快捷输入配置。
  - 首次安装默认会带出 7 个内置快捷输入：`概括 / 缩写 / 重构 / 反直觉 / 审计 / 第一性原理 / 意图分析`。
  - 采用高密度、多行换行布局，不假设单行可容纳全部 promptTab。
  - 标签状态优先使用 loading icon、完成态 icon 和失败态 icon 表达，不把状态文案铺满标签正文。
- 聊天与输入区：
  - 聊天区位于 `promptTab` 区下方，展示当前 `promptTab` 对应的消息、分支和局部操作。
  - 输入区固定在底部，包含文本输入、模型选择、发送、“包含页面内容”、当前 `promptTab` 清空/停止、导出。
  - 输入区高度可拖拽，布局变化不能遮挡关键按钮。
  - 消息级编辑、重试、复制、定位顶部/底部、继续新增分支、停止分支、删除分支统一收敛为紧凑局部按钮，不让正文区退化成第二层工具栏。

- 当前编辑与重试的上下文重建仅复用持久化后的用户消息，不会回放历史请求级 `pageContent` 注入。

## 2.10 快捷输入当前行为

- 快捷输入不再把 `prompt` 预填为输入框草稿；输入框只保留用户手动输入的真实草稿。
- 打开侧边栏时，若页面已存在缓存正文，则仅对 `autoTrigger=true` 且当前无历史、无 loading 的快捷输入直接发起请求。
- 点击快捷输入 `promptTab` 时，若当前标签无历史、无 loading，则直接发送该快捷输入的真实 `prompt`，而不是把它写进输入框。
- 快捷输入自动发送统一强制携带页面正文，仍作为请求级 override，不改单页持久化的 `includePageContent` 语义。
- 自动触发产生的用户消息正文仍保存真实 `prompt`，但 UI 默认展示快捷输入名称；进入编辑态时展示真实 `prompt`，编辑保存后展示编辑后的文本。

## 4. 能力边界

负责：

- 展示当前页面信息和提取内容。
- 处理顶部工具按钮，包括切换提取方式、复制提取内容、重新提取、清空当前页面数据、打开历史页、打开设置页和打开 GitHub。
- 展示并切换 `promptTab`。
- 在 side panel 初始化完成后主动调用 `GET_SIDEBAR_BOOTSTRAP`，先恢复缓存和状态，再按需进入黑名单确认、提取和自动触发。
- 保持提取内容区常驻显示，并与 `promptTab` 聊天区并存。
- 发起发送、停止当前 `promptTab` 会话、清空当前 `promptTab` 会话、导出。
- 展示主回答与分支回答。
- 发起消息编辑、消息重试、消息复制、消息定位、分支继续新增、分支停止和分支删除。
- 呈现黑名单确认层和提取错误态。

不负责：

- 直接访问远端模型或同步后端。
- 直接维护持久化 key。
- 直接决定 MV3 权限和脚本注入策略。

## 6. 关键流程

- 打开侧边栏并优先展示缓存。
- 打开侧边栏后先恢复，再进入受控初始化。
- `browserTab` 切换后的 side panel 自动隐藏与再次进入。
- 顶部控制区按钮操作。
- 内容提取与方法切换。
- 提取区复制与高度拖拽。
- 页面级清空。
- 文本/图片发送。
- `Enter` 发送、`Shift + Enter` 换行。
- 自动触发快捷输入与去重。
- 分支并发与预览。
- 输入区高度拖拽与布局切换。
- Markdown 渲染、消息编辑、重试、复制和分支局部操作。
- 多分支横向阅读列与窄屏回落。
- 侧边栏关闭重开后的 loading 恢复。

## 7. 错误与异常处理

- content script 未连接时，先自动刷新当前页面一次并重试，仍失败后再提示用户手动重试。
- 提取失败时保留旧内容和当前页面上下文。
- 流式错误时回收 loading 并标记目标消息或分支。
- 用户取消不显示系统错误。
- 黑名单命中时先阻断自动流程。
- 默认黑名单当前会优先拦截 Google / Bing / 百度搜索结果页。
- 黑名单放行前不允许开始提取或自动触发。
- side panel 再次打开时，有缓存不重复提取；已有历史或 loading 的 `promptTab` 不重复自动触发。
- 快捷输入 `promptTab` 首次进入时若当前无历史，会直接触发请求，不再依赖预填草稿后再手动发送。
- 自动触发完成后的 `done/error` 标签文案当前通过下一次 bootstrap 恢复；首次打开期间主要依赖流事件和消息落库结果。
- 标签和消息状态的当前 UI 约束是“文字降级、图标优先”：正文区只保留必要提示，持续态优先由动画和状态图标表达。

## 10. 约束与禁止事项

- 不能在 UI 直接使用 provider SDK。
- 不能假设 side panel 始终处于打开状态。
- 不能通过 DOM 状态反推持久化事实。
- 不能把自动触发设计成强制切换当前用户视图。
- 自动触发若需要强制带入页面内容，只能作为请求级 override，不能改写页面级 `includePageContent`。
- 分支模型候选只能来自当前启用且配置完整的模型。
- 提取内容区必须常驻显示，不能因切换 `promptTab`、发送消息或分支加载而折叠隐藏。
- `promptTab` 区必须支持多行换行，不能要求用户横向滚动才能访问常用 `promptTab`。
- 顶部控制区和底部输入区必须保持稳定可达，不能随主阅读区滚动离开视口。
- 顶部控制区按钮不能破坏当前 `promptTab`、输入内容和滚动上下文。
- 页面级和消息级动作默认使用 icon button；如果必须展示文字，只能用于输入确认类动作，不用于工具栏。
- 页面级清空与 `promptTab` 级清空不能复用同一语义或同一确认文案。
- 清空当前 `promptTab` 会话时必须同步重置该 `promptTab` 的自动触发初始化状态。
- 清空当前页面数据时必须同时清理页面缓存、页面级状态和该页面下的 `promptTab` 会话数据。
- `browserTab` 切换后 side panel 允许自动隐藏，但切回原 `browserTab` 时不允许自动恢复。
