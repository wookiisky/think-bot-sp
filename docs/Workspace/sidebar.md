# 侧边栏

## 1. 模块定位

侧边栏是“当前网页即时处理”的主工作台，负责页面提取、快速提问、快捷输入、分支分析和导出。

术语约定：

- `browserTab`：Chrome 浏览器标签页。
- `promptTab`：侧边栏内的 `Chat` 或某个快捷输入标签。

## 2. 界面布局映射

侧边栏界面按设计图拆成 4 个稳定区域：

- 顶部控制区：
  - 固定在顶部。
  - 左侧是提取相关控件：`Readability`、`Jina`、复制提取内容、重新提取。
  - 右侧是页面级和系统级控件：清空当前页面数据、打开历史页、打开设置页、打开 GitHub。
  - 控件视觉统一采用 icon button，动作语义通过 `aria-label`、即时显示的自定义 tooltip 和 i18n 资源提供，不再依赖原生 `title`。
  - 破坏性动作统一通过按钮旁侧的 mini confirm 二次确认，不使用原生 `window.confirm`。
  - 不再展示单独的 `browserTab` 标识行，顶部次级信息只保留页面级 notice。
- 提取内容区：
  - 作为常驻独立区域显示，不因切换 `promptTab` 而隐藏。
  - 位于顶部控制区下方、`promptTab` 区上方，承担“当前页面正文阅读区”职责。
  - `Readability` 结果统一转成 Markdown 后持久化，`Jina` 保持现有文本链路。
  - 正文区展示原始 Markdown 纯文本，不再渲染标题、列表、链接等富文本结构；支持正常、加载中、提取失败三种状态，支持复制与垂直拖拽调整高度。
  - 页面级反馈统一通过顶部 toast 提示，不再在提取区下方铺一行 badge notice。
- 快捷输入标签区：
  - 位于提取内容区下方。
  - 默认包含 `Chat` promptTab，其余 promptTab 来自快捷输入配置。
  - 首次安装默认会带出 7 个内置快捷输入：`概括 / 缩写 / 重构 / 反直觉 / 审计 / 第一性原理 / 意图分析`。
  - 采用高密度、多行换行布局，不假设单行可容纳全部 promptTab。
  - 标签宽度按名称自适应，不再使用固定最小宽度。
  - 标签状态优先使用纯视觉装饰：已有正文显示底部彩色线，loading 显示四周旋转彩色边框，不把状态文案铺满标签正文。
- 聊天与输入区：
  - 聊天区位于 `promptTab` 区下方，展示当前 `promptTab` 对应的消息、分支和局部操作。
  - 输入区固定在底部，改为两行：首行单行输入框，图片按钮和发送按钮位于输入框右侧；第二行放模型选择、“包含页面内容”、清空、导出。
  - 输入区高度可拖拽，拖拽 handle 与提取区使用同一视觉样式，继续保持本地运行态，但占位更小。
  - 消息区不再展示角色名称，也不再使用带边框卡片，改为消息间横线分隔。
  - 助手生成中时只保留消息开头的 loading 圈，不再额外占用一整行状态栏。
  - 助手生成中的停止动作直接放到该 loading 消息内部，不再放在输入区第二行。
- 用户消息保留消息级编辑、重试与复制按钮；助手输出统一按“分支消息卡片”展示，不再区分主回答卡片与分支卡片两套入口。
- 助手消息当前统一按分支卡片展示；单分支展示 1 列，2 个分支时固定为 2 列并由外层容器承载横向滚动，超过 2 个分支后所有列统一保持 `350px` 最小宽度。
  - 水平滚动条固定由分支区外层容器承载，避免落到单个分支卡片内部。
  - 多列分支区同时提供顶部和底部两条同步的水平滚动条，作用一致。
  - 消息按钮栏显示在对应消息内部，采用“外层绝对定位 + 内层 sticky”固定在消息右侧中线附近，消息纵向经过视口时仍尽量保持可见。
  - 图标按钮当前统一约束为“hover / focus 立即显示 tooltip”，并保留 `aria-label` 作为可访问名称。

## 2.10 快捷输入当前行为

- 快捷输入不再把 `prompt` 预填为输入框草稿；输入框只保留用户手动输入的真实草稿。
- 打开侧边栏时，若页面已存在缓存正文，则仅对 `autoTrigger=true` 且当前无历史、无 loading 的快捷输入直接发起请求。
- 点击快捷输入 `promptTab` 时，若当前标签无历史、无 loading，则直接发送该快捷输入的真实 `prompt`，而不是把它写进输入框。
- 快捷输入自动发送统一强制携带页面正文，仍作为请求级 override，不改单页持久化的 `includePageContent` 语义。
- 自动触发产生的用户消息正文仍保存真实 `prompt`，但 UI 默认展示快捷输入名称；进入编辑态时展示真实 `prompt`，编辑保存后展示编辑后的文本。
- 首轮快捷输入若请求失败，只保留当前打开 side panel 期间的错误提示，不持久化用户消息、助手错误态和 `auto-error` 标签状态；再次打开时会重新触发。

## 4. 能力边界

负责：

- 展示当前页面信息和提取内容。
- 处理顶部工具按钮，包括切换提取方式、复制提取内容、重新提取、清空当前页面数据、打开历史页、打开设置页和打开 GitHub。
- 展示并切换 `promptTab`。
- 在 side panel 初始化完成后主动调用 `GET_SIDEBAR_BOOTSTRAP`，先恢复缓存和状态，再按需进入黑名单确认、提取和自动触发。
- 保持提取内容区常驻显示，并与 `promptTab` 聊天区并存。
- 发起发送、停止当前 `promptTab` 会话、清空当前 `promptTab` 会话、导出。
- 展示主回答与分支回答。
- 发起消息编辑、消息重试、消息复制、消息定位、分支继续新增、分支停止、分支删除和分支独立预览。
- 呈现黑名单确认层和提取错误态。

不负责：

- 直接访问远端模型或同步后端。
- 直接维护持久化 key。
- 直接决定 MV3 权限和脚本注入策略。

## 7. 错误与异常处理

- content script 未连接时，先自动刷新当前页面一次并重试，仍失败后再提示用户手动重试。
- 提取失败时保留旧内容和当前页面上下文。
- 流式错误时回收 loading 并标记目标消息或分支；若 provider 返回明确错误文本，直接显示该文本。
- 首轮快捷输入失败时回滚该轮新增消息，只保留当前界面的错误提示，避免把失败首轮误判成已有历史。
- 用户取消不显示系统错误。
- 黑名单命中时先阻断自动流程。
- 默认黑名单当前会优先拦截 Google / Bing / 百度搜索结果页。
- 黑名单放行前不允许开始提取或自动触发。
- side panel 再次打开时，有缓存不重复提取；已有历史或 loading 的 `promptTab` 不重复自动触发。
- 快捷输入 `promptTab` 首次进入时若当前无历史，会直接触发请求，不再依赖预填草稿后再手动发送。
- 点击快捷输入时，若提取区当前为空，会先自动提取，再触发该快捷输入；提取失败则只提示错误，不发送聊天。
- 自动触发完成后的 `done/error` 标签文案当前通过下一次 bootstrap 恢复；首次打开期间主要依赖流事件和消息落库结果。
- 标签和消息状态的当前 UI 约束是“文字降级、图标优先”：正文区只保留必要提示，持续态优先由动画和状态图标表达。
- `Readability` Markdown 当前只保证原始文本可读性，不承诺在提取区保留标题层级、列表语义、复杂表格、嵌入式组件或脚本交互结构。
- 当前视觉约束统一为“全局无圆角”，局部组件即使带 `rounded-*` 也会被基础样式覆盖。
- 分支预览层当前复用聊天区的同一套 Markdown 渲染器，不额外维护独立渲染语义。
- 删除最后一个分支时，会直接删除整条助手消息，避免在历史中留下空助手壳记录。

## 10. 约束与禁止事项

- 不能在 UI 直接使用 provider SDK。
- 不能假设 side panel 始终处于打开状态。
- 不能通过 DOM 状态反推持久化事实。
- 不能把自动触发设计成强制切换当前用户视图。
- 自动触发若需要强制带入页面内容，只能作为请求级 override，不能改写页面级 `includePageContent`。
- 手动新增分支的模型候选只能来自当前启用且配置完整的模型，并包含主模型。
- 提取内容区必须常驻显示，不能因切换 `promptTab`、发送消息或分支加载而折叠隐藏。
- `promptTab` 区必须支持多行换行，不能要求用户横向滚动才能访问常用 `promptTab`。
- 顶部控制区和底部输入区必须保持稳定可达，不能随主阅读区滚动离开视口。
- 顶部控制区按钮不能破坏当前 `promptTab`、输入内容和滚动上下文。
- 页面级和消息级动作默认使用 icon button；如果必须展示文字，只能用于输入确认类动作，不用于工具栏。
- 用户消息和助手消息的层次区分必须依赖背景与间距，不能重新引入显式分割线。
- 页面级清空与 `promptTab` 级清空不能复用同一语义或同一确认文案。
- 清空当前 `promptTab` 会话时必须同步重置该 `promptTab` 的自动触发初始化状态。
- 清空当前页面数据时必须同时清理页面缓存、页面级状态和该页面下的 `promptTab` 会话数据。
- `browserTab` 切换后 side panel 允许自动隐藏，但切回原 `browserTab` 时不允许自动恢复。

## 11. 测试要求

- 职责测试：打开侧边栏、提取、发送、分支、导出。
- 职责测试：顶部控制区按钮、提取区复制、图片预览、拖拽调整。
- 职责测试：提取内容区常驻显示，且与聊天区并存。
- 职责测试：每个分支卡片都有独立悬浮按钮组，且“新增分支”命令成功后立即出现 loading 分支卡片。
- 职责测试：删除最后一个分支时整条助手消息一起移除。
- 边界测试：受限页退化、黑名单拦截、配置不完整模型禁止发送。
- 正常流测试：文本、图片、快捷输入、多 `promptTab` 切换、自动触发、`promptTab` 多行换行。

- 不变量测试：页面级 `includePageContent` 恢复、`promptTab` 隔离、自动触发去重。
- 可观测性测试：关键 loading 和错误事件可见。
