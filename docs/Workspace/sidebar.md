# 侧边栏

## 1. 模块定位

侧边栏是“当前网页即时处理”的主工作台，负责页面提取、快速提问、快捷输入、分支分析和导出。

术语约定：

- `browserTab`：Chrome 浏览器标签页。
- `promptTab`：侧边栏内的 `Chat` 或某个快捷输入标签。

## 2. 界面布局映射

侧边栏界面按设计图拆成 4 个稳定区域：

- 顶部控制区：
  - 固定在顶部。
  - 左侧是提取相关控件：`Readability`、`Jina`、复制提取内容、重新提取。
  - 右侧是页面级和系统级控件：清空当前页面数据、打开历史页、打开设置页、打开 GitHub。
- 快捷输入标签区：
  - 位于顶部控制区下方。
  - 默认包含 `Chat` promptTab，其余 promptTab 来自快捷输入配置。
  - 采用高密度、多行换行布局，不假设单行可容纳全部 promptTab。
- 提取内容区：
  - 作为常驻独立区域显示，不因切换 `promptTab` 而隐藏。
  - 位于 promptTab 区与聊天区之间，承担“当前页面正文阅读区”职责。
  - 支持正常、加载中、提取失败三种状态，支持复制与垂直拖拽调整高度。
- 聊天与输入区：
  - 聊天区位于提取内容区下方，展示当前 `promptTab` 对应的消息、分支和局部操作。
  - 输入区固定在底部，包含文本输入、模型选择、发送、“包含页面内容”、当前 `promptTab` 清空/停止、导出。
  - 输入区高度可拖拽，布局变化不能遮挡关键按钮。

## 3. 核心抽象

- 图片预览区

## 4. 能力边界

负责：

- 展示当前页面信息和提取内容。
- 处理顶部工具按钮，包括切换提取方式、复制提取内容、重新提取、清空当前页面数据、打开历史页、打开设置页和打开 GitHub。
- 展示并切换 `promptTab`。
- 在 side panel 初始化完成后主动调用 `GET_PAGE_INFO`，优先展示缓存并按需触发提取。
- 保持提取内容区常驻显示，并与 `promptTab` 聊天区并存。
- 发起发送、停止当前 `promptTab` 会话、清空当前 `promptTab` 会话、导出。
- 展示主回答与分支回答。
- 发起消息编辑、消息重试、分支继续新增、分支停止和分支删除。
- 处理提取区和输入区的拖拽高度调整。
- 呈现黑名单确认层和提取错误态。

不负责：

- 直接访问远端模型或同步后端。
- 直接维护持久化 key。
- 直接决定 MV3 权限和脚本注入策略。

## 5. 对外接口

- one-shot command：
  - 获取页面信息
  - 确认黑名单放行
  - 切换提取方式
  - 重新提取
  - 清空当前页面数据
  - 发送消息
  - 停止会话
  - 停止分支
  - 删除分支
  - 编辑用户消息
  - 重试消息
  - 继续新增分支
  - 清空当前 `promptTab` 会话
  - 导出对话
- long-lived port：
  - 订阅流式 chunk、完成、错误、loading 更新

## 6. 关键流程

- 打开侧边栏并优先展示缓存。
- `browserTab` 切换后的 side panel 自动隐藏与再次进入。
- 顶部控制区按钮操作。
- 内容提取与方法切换。
- 提取区复制与高度拖拽。
- 页面级清空与 `promptTab` 级清空。
- 文本/图片发送。
- 图片预览添加与移除。
- 自动触发快捷输入与去重。
- 分支并发与预览。
- 输入区高度拖拽与布局切换。
- 消息编辑、重试和分支局部操作。
- 侧边栏关闭重开后的 loading 恢复。

## 7. 错误与异常处理

- content script 未连接时，先自动刷新当前页面一次并重试，仍失败后再提示用户手动重试。
- 提取失败时保留旧内容和当前页面上下文。
- 流式错误时回收 loading 并标记目标消息或分支。
- 用户取消不显示系统错误。
- 黑名单命中时先阻断自动流程。
- side panel 再次打开时，有缓存不重复提取；已有历史或 loading 的 `promptTab` 不重复自动触发。

## 8. 数据与状态

- UI 运行态：
  - 当前提取内容显示状态
  - 当前提取方式
  - 当前活动 `promptTab`
  - `promptTab` 矩阵换行结果
  - 当前提取区高度
  - 提取内容区滚动位置
  - 当前聊天区滚动位置
  - 输入框内容
  - 当前输入区高度
  - 本地预览图片
  - 分支预览层显示状态
  - 弹层显示状态
- 页面级持久化辅助状态：
  - `promptTab` 初始化状态
  - `promptTab` 自动触发状态
- 持久化依赖：
  - `PageRecord`
  - `ConversationRecord`
  - `LoadingStateRecord`
  - `ExtensionConfig`

页面级动作与 `promptTab` 级动作必须严格分离：

- 页面级动作：
  - 切换提取方式
  - 重新提取
  - 清空当前页面数据
- `promptTab` 级动作：
  - 发送消息
  - 停止当前 `promptTab` 会话
  - 清空当前 `promptTab` 会话
  - 导出当前 `promptTab` 对话

## 10. 约束与禁止事项

- 不能在 UI 直接使用 provider SDK。
- 不能假设 side panel 始终处于打开状态。
- 不能通过 DOM 状态反推持久化事实。
- 不能把自动触发设计成强制切换当前用户视图。
- 提取内容区必须常驻显示，不能因切换 `promptTab`、发送消息或分支加载而折叠隐藏。
- `promptTab` 区必须支持多行换行，不能要求用户横向滚动才能访问常用 `promptTab`。
- 顶部控制区和底部输入区必须保持稳定可达，不能随主阅读区滚动离开视口。
- 顶部控制区按钮不能破坏当前 `promptTab`、输入内容和滚动上下文。
- 页面级清空与 `promptTab` 级清空不能复用同一语义或同一确认文案。
- 清空当前 `promptTab` 会话时必须同步重置该 `promptTab` 的自动触发初始化状态。
- 清空当前页面数据时必须同时清理页面缓存、页面级状态和该页面下的 `promptTab` 会话数据。
- `browserTab` 切换后 side panel 允许自动隐藏，但切回原 `browserTab` 时不允许自动恢复。

## 11. 测试要求

- 职责测试：打开侧边栏、提取、发送、分支、导出。
- 职责测试：顶部控制区按钮、提取区复制、图片预览、拖拽调整。
- 职责测试：提取内容区常驻显示，且与聊天区并存。
- 边界测试：受限页退化、黑名单拦截、配置不完整模型禁止发送。
- 正常流测试：文本、图片、快捷输入、自动触发、`promptTab` 多行换行。
- 错误流测试：提取失败、流式错误、content script 未连接。
- 异常流测试：side panel 重开恢复、`browserTab` 切换隐藏后重新进入、取消请求、分支局部删除、编辑消息后重发、页面级清空与 `promptTab` 级清空。
- 不变量测试：页面级 `includePageContent` 恢复、`promptTab` 隔离、自动触发去重。
- 可观测性测试：关键 loading 和错误事件可见。
