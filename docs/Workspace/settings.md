# 设置页

## 1. 模块定位

设置页是长期配置的统一查看与局部编辑入口。阶段2当前交付模型编辑、语言与主题即时预览、恢复默认、导入导出、本地缓存统计与安全清理，以及快捷输入预览。

## 2. 核心抽象

- 基础设置
- 标签页预览
- 语言模型
- 云同步占位
- 黑名单占位
- 顶部全局操作

## 3. 能力边界

负责：

- 展示当前阶段已交付的配置子集，并对这些输入做结构校验。
- 触发保存、恢复默认、导入导出，并展示后续阶段预留的同步入口。
- 即时预览语言和主题。
- 展示本地缓存占用并执行本地缓存清理。
- 展示快捷输入预览列表。
- 展示最近一次错误信息。
- 阶段2只保证当前已交付字段的本地闭环，不负责远端同步的实际执行。

不负责：

- 直接执行模型请求。
- 直接写远端同步文件。
- 直接操作历史页面或聊天记录。
- 直接把保存流程扩展成远端同步执行链路。

## 4. 对外接口

- 获取完整配置。
- 保存当前表单生成的完整配置。
- 执行导入导出。

## 5. 关键流程

- 页面采用左侧 5 个栏目加顶部常驻操作区的布局：
  - 左侧栏目固定为“基础设置 / 标签页 / 语言模型 / 云同步 / 黑名单设置”。
  - 阶段2顶部当前交付“保存 / 恢复默认 / 导出配置 / 导入配置”；不交付“保存并同步”入口。
- 语言模型采用“列表摘要 + 单项展开编辑”的交互：
  - 列表行展示显示名称、Provider、模型标识和启用状态。
  - 阶段2当前只交付单项切换和展开编辑，不交付新增、复制、删除、拖拽排序。
  - 展开后按 Provider 展示差异字段，覆盖 Base URL、API Key、Model、Deployment、Temperature、Thinking Budget、最大输出 token。
  - API Key 默认掩码显示，允许用户临时查看明文。
- 快捷输入区域在阶段2只展示过滤后的只读预览，不交付编辑、排序和远端模板导入。
- 语言和主题切换都支持即时预览并在保存后持久化。
- 云同步和黑名单在阶段2只保留导航占位，不交付表单编辑。
- 基础设置展示当前本地缓存占用，并允许清理可安全回收的本地缓存数据。
- 阶段2已交付本地设置闭环，远端同步执行留到后续阶段单独实现。

## 6. 错误与异常处理

- 保存失败时保留当前表单。
- 导入失败时不覆盖现有配置。
- 导出失败时保留当前页面状态，并把错误展示在主视图。
- 本地缓存清理失败时保留当前占用统计并展示错误，不影响其他配置编辑。
- 阶段2不提供“保存并同步”和连接测试执行链路，不承诺已完成远端同步落盘。

## 7. 数据与状态

- UI 运行态：
  - 当前导航分栏
  - 当前展开的模型项
  - 本地缓存占用统计
  - 未保存变更标记
  - 表单校验错误
  - 最近一次错误展示
- 持久化依赖：
  - `ExtensionConfig`
  - `SyncSnapshot.lastSyncAt`

## 9. 约束与禁止事项

- 左侧栏目切换不能丢未保存表单。
- 左侧栏目数量和顺序必须与产品稿一致，不在导航区单独增加“导入导出”栏目。
- 默认模型候选只允许“启用且配置完整”的模型。
- “启用且配置完整”的模型定义在设置页、侧边栏和对话管理页必须一致。
- Provider 切换后，表单字段显隐和校验规则必须同步切换，不能保留上一个 Provider 的无效必填约束。
- 国际化只允许使用平铺 key。
- 图标资源不能通过在线字体加载。
- 阶段2导出动作必须把配置内容真正交给用户，而不是只调用后台命令。
- 恢复默认必须通过 background 命令链路执行，不能由 UI 本地拼装默认值代替真实写回。

## 10. 测试要求

- 职责测试：模型、快捷输入预览、语言切换、主题切换、本地缓存、恢复默认、导入导出。
- 职责测试：本地缓存占用展示与缓存清理。
- 边界测试：不完整模型、重复 ID、非法规则。
- 正常流测试：保存、恢复默认、语言切换、主题切换、模型展开编辑、快捷输入预览、导出配置、导入恢复、本地缓存清理。
- 错误流测试：保存失败、导入失败、导出失败、缓存清理失败、background 命令返回业务错误。
- 异常流测试：快速重复保存、切换栏目后保持表单、Provider 切换后字段显隐恢复正确。
- 不变量测试：软删除配置项不破坏历史引用。
- 可观测性测试：错误信息、API Key 掩码状态展示。

## 12. 阶段 2.5 设计系统基线

- 设置页统一复用 `assets/styles/globals.css` 提供的 Tailwind CSS v4 主题 token。
- 外层壳层和信息卡片统一基于 `src/components/ui/card.tsx`、`src/components/ui/button.tsx`、`src/components/ui/separator.tsx` 组合。
- 模型编辑表单统一复用 `src/components/ui/input.tsx`、`src/components/ui/badge.tsx`，并保持主题切换后的即时预览。
- 快捷输入预览区统一复用 `Card + Button` 基础件，不再单独维护一套壳层样式。
- 阶段 2.5 只统一基础布局和控件来源，不改变设置页命令链路、日志点和本地配置闭环语义。
