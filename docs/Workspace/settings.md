# 设置页

## 1. 模块定位

设置页是长期配置的统一查看与局部编辑入口。阶段2当前交付模型编辑、语言与主题即时预览、恢复默认、导入导出、本地缓存统计与安全清理、快捷输入配置编辑，以及手动云同步配置与执行。
设置页以独立完整 tab 页面展示，不使用弹窗式居中大卡片作为整页壳层。

## 2. 核心抽象

- 基础设置
- 标签页预览
- 语言模型
- 云同步
- 黑名单占位
- 顶部全局操作

## 3. 能力边界

负责：

- 展示当前阶段已交付的配置子集，并对这些输入做结构校验。
- 触发保存、恢复默认、导入导出，并执行手动同步相关动作。
- 即时预览语言和主题。
- 展示本地缓存占用并执行本地缓存清理。
- 展示并编辑快捷输入配置列表。
- 编辑同步 provider 配置，执行连接测试与手动同步。
- 展示最近一次错误信息。
- 阶段2只保证配置级别的手动同步闭环，不负责页面、会话和墓碑同步。

不负责：

- 直接执行模型请求。
- 绕过 background 直接写远端同步文件。
- 直接操作历史页面或聊天记录。
- 自动拉取远端并做对象级冲突合并。
- 把保存流程扩展成“保存并同步”一体化链路。

## 4. 对外接口

- 获取完整配置。
- 保存当前表单生成的完整配置。
- 执行导入导出。
- 测试当前同步配置连通性。
- 手动把当前完整配置同步到远端，并回写 `lastSyncAt`。
- 所有“打开设置页”入口统一复用设置页导航能力，并最终调用 `chrome.runtime.openOptionsPage()`。
- WXT 项目里设置页是否独立 tab 打开，必须通过 `entrypoints/options/index.html` 里的 `<meta name="manifest.open_in_tab" content="true" />` 控制，不能在 `wxt.config.ts` 里手写 `options_ui/options_page`。

## 5. 关键流程

- 页面采用左侧 5 个栏目加顶部常驻操作区的布局：
  - 左侧栏目固定为“基础设置 / 标签页 / 语言模型 / 云同步 / 黑名单设置”。
  - 阶段2顶部当前交付“保存 / 恢复默认 / 导出配置 / 导入配置”；同步动作收口在“云同步”分栏，不交付“保存并同步”入口。
  - 页面整体使用完整 tab 布局，顶部信息区和内容区直接铺在页面中，内部功能块仍使用卡片承载。
- 语言模型采用“列表摘要 + 单项展开编辑”的交互：
  - 列表行展示显示名称、Provider、模型标识和启用状态。
  - 当前已交付新增、复制、软删除和顺序调整按钮；顺序调整先以基础按钮实现，不要求拖拽。
  - 展开后按 Provider 展示差异字段，覆盖 Name、Provider、Enabled、Base URL、API Key、Model、Deployment、Temperature、Thinking Budget、最大输出 token、Tools、`supportsImages`。
  - API Key 默认掩码显示，允许用户临时查看明文。
- 快捷输入区域当前交付列表编辑、单项表单、软删除和顺序调整。
- 快捷输入当前可编辑 `name / prompt / autoTrigger / modelId / order / deletedAt`。
- 快捷输入当前不交付远端模板导入，也不把自动触发执行链路纳入设置页闭环。
- 语言和主题切换都支持即时预览并在保存后持久化。
- 云同步分栏当前交付 `enabled / provider / gist / webdav / 测试连接 / 立即同步 / 最近同步时间`。
- 手动同步当前只推送配置快照，不拉取远端，不合并页面、会话和墓碑。
- 黑名单在阶段2只保留导航占位，不交付表单编辑。
- 基础设置展示当前本地缓存占用，并允许清理可安全回收的本地缓存数据。
- 阶段2已交付本地设置闭环与最小同步闭环，远端合并、删除传播和自动同步留到后续阶段单独实现。

## 6. 错误与异常处理

- 保存失败时保留当前表单。
- 导入失败时不覆盖现有配置。
- 导出失败时保留当前页面状态，并把错误展示在主视图。
- 本地缓存清理失败时保留当前占用统计并展示错误，不影响其他配置编辑。
- 连接测试失败时保留当前同步表单并展示错误反馈。
- 手动同步失败时保留当前草稿和最近一次成功同步时间，不把失败结果写成成功状态。
- 阶段2不提供“保存并同步”、远端拉取或冲突合并，不承诺页面和会话数据已进入同步闭环。

## 7. 数据与状态

- UI 运行态：
  - 当前导航分栏
  - 已保存配置快照
  - 当前草稿配置
  - 当前展开的模型项
  - 本地缓存占用统计
  - 未保存变更标记
  - 同步测试进行态
  - 同步执行进行态
  - 最近一次同步反馈
  - 表单校验错误
  - 最近一次错误展示
- 持久化依赖：
  - `ExtensionConfig`
  - `SyncSnapshot.lastSyncAt`

## 9. 约束与禁止事项

- 左侧栏目切换不能丢未保存表单。
- 左侧栏目数量和顺序必须与产品稿一致，不在导航区单独增加“导入导出”栏目。
- 设置页必须保持完整 tab 页面观感，不能回退成弹窗式整页包裹卡片。
- 默认模型候选只允许“启用且配置完整”的模型。
- “启用且配置完整”的模型定义在设置页、侧边栏和对话管理页必须一致。
- 图片输入能力必须通过 `supportsImages` 显式建模，不允许由 UI 或调度层按模型名猜测。
- Provider 切换后，表单字段显隐和校验规则必须同步切换，不能保留上一个 Provider 的无效必填约束。
- `TEST_SYNC_CONNECTION` 与 `SYNC_NOW` 必须走 background 命令链路，不允许 options 页面直接发远端请求。
- `SYNC_NOW` 必须先保存当前配置，再把成功结果回写到 `sync.lastSyncAt`。
- 国际化只允许使用平铺 key。
- 图标资源不能通过在线字体加载。
- 阶段2导出动作必须把配置内容真正交给用户，而不是只调用后台命令。
- 恢复默认必须通过 background 命令链路执行，不能由 UI 本地拼装默认值代替真实写回。

## 10. 测试要求

- 职责测试：模型、快捷输入编辑、语言切换、主题切换、本地缓存、恢复默认、导入导出、同步连接测试与手动同步。
- 职责测试：本地缓存占用展示与缓存清理。
- 边界测试：不完整模型、重复 ID、非法规则。
- 正常流测试：保存、恢复默认、语言切换、主题切换、模型新增、复制、软删除、顺序调整、展开编辑、快捷输入新增、编辑、软删除、顺序调整、导出配置、导入恢复、本地缓存清理、同步连接测试、手动同步。
- 错误流测试：保存失败、导入失败、导出失败、缓存清理失败、同步连接失败、手动同步失败、background 命令返回业务错误。
- 异常流测试：快速重复保存、切换栏目后保持表单、Provider 切换后字段显隐恢复正确、service worker 启动后再注入测试 provider 仍可命中同步桩。
- 不变量测试：软删除配置项不破坏历史引用。
- 可观测性测试：错误信息、API Key 掩码状态展示。
