# 设置页

## 1. 模块定位

设置页是长期配置的统一查看与局部编辑入口。当前已交付模型编辑、语言与主题即时预览、恢复默认、文件导入导出、本地缓存统计与安全清理、提取默认参数配置、快捷输入配置编辑、黑名单配置编辑，以及手动云同步与“保存并同步”动作。
设置页以独立完整 tab 页面展示，不使用弹窗式居中大卡片作为整页壳层。

## 2. 核心抽象

- 基础设置
- 快捷输入
- 语言模型
- 云同步
- 黑名单设置
- 顶部全局操作

## 3. 能力边界

负责：

- 展示当前阶段已交付的配置子集，并对这些输入做结构校验。
- 触发保存、保存并同步、恢复默认、导入导出，并执行手动同步相关动作。
- 即时预览语言和主题。
- 展示本地缓存占用并执行本地缓存清理。
- 展示并编辑快捷输入配置列表，包括内置默认模板。
- 编辑同步 provider 配置，执行连接测试与手动同步。
- 展示并编辑黑名单规则，支持规则测试与恢复默认规则。
- 阶段2只保证配置级别的手动同步闭环，不负责页面、会话和墓碑同步。

不负责：

- 直接执行模型请求。
- 绕过 background 直接写远端同步文件。
- 直接操作历史页面或聊天记录。
- 自动拉取远端并做对象级冲突合并。

## 4. 对外接口

- 获取完整配置。
- 保存当前表单生成的完整配置。
- 保存当前表单并立即执行一次手动同步。
- 执行导入导出。
- 测试当前同步配置连通性。
- 手动把当前完整配置同步到远端，并回写 `lastSyncAt`。
- 所有“打开设置页”入口统一复用设置页导航能力，并最终调用 `chrome.runtime.openOptionsPage()`。
- WXT 项目里设置页是否独立 tab 打开，必须通过 `entrypoints/options/index.html` 里的 `<meta name="manifest.open_in_tab" content="true" />` 控制，不能在 `wxt.config.ts` 里手写 `options_ui/options_page`。

## 5. 关键流程

- 页面采用左侧 5 个栏目加顶部常驻操作区的布局：
  - 左侧栏目固定为“基础设置 / 快捷输入 / 语言模型 / 云同步 / 黑名单设置”。
  - 顶部当前交付“保存 / 保存并同步 / 恢复默认 / 导入配置 / 导出配置”。
  - 页面整体使用完整 tab 布局，顶部信息区和内容区直接铺在页面中，内部功能块仍使用卡片承载。
- 语言模型采用“列表摘要 + 单项展开编辑”的交互：
  - 模型项标题栏单行展示显示名称，小字展示 `Provider / 模型标识`。
  - 标题栏右侧统一提供 `复制 / 上移 / 下移 / 删除 / 启用`；其中 `复制 / 上移 / 下移 / 删除` 当前采用 icon button + 即时 tooltip。
  - 当前已交付新增、复制、软删除、拖拽排序和顺序调整按钮；“新增模型”位于卡片标题栏右侧，“复制模型”位于单个模型标题栏中；拖拽与上下移动回退方案同时保留。
  - 模型列表允许全部收起，不强制保留一个常驻展开项。
  - 展开后的模型表单按两列排布，长字段跨整行展示。
  - 展开后按 Provider 展示差异字段，覆盖 Name、Provider、Base URL、API Key、Model、Deployment、Temperature、Thinking Budget、最大输出 token、Tools、`supportsImages`。
  - API Key 默认掩码显示，允许用户临时查看明文。
- 快捷输入区域当前交付列表编辑、内联折叠展开、消息预览、拖拽排序、软删除和顺序调整。
  - 快捷输入项标题栏单行展示名称，小字展示提示词预览。
  - 标题栏右侧统一提供 `上移 / 下移 / 删除 / 自动触发`；其中 `上移 / 下移 / 删除` 当前采用 icon button + 即时 tooltip。
- 首次安装时默认写入 7 个内置快捷输入：`概括 / 缩写 / 重构 / 反直觉 / 审计 / 第一性原理 / 意图分析`。
- 旧配置读取与导入时采用“补缺不覆盖”：只补齐缺失的内置快捷输入，不覆盖已有同 id 项。
- 快捷输入当前可编辑 `name / prompt / autoTrigger / modelId / parallelModelIds / order / deletedAt`。
- 快捷输入支持从公开 `quick_input_tabs.json` 导入远端模板：
  - 导入时为模板重建本地 ID。
  - 按 `name + prompt` 跳过已存在项，只追加新模板。
- 失效 `modelId` 会降级成 `null`，失效 `parallelModelIds` 会在导入时过滤。
- 基础设置当前交付默认提取方式、默认提取区高度、Jina API Key、Jina 响应模板和“默认附带页面正文”。
- 基础设置和快捷输入中的并行模型选择统一使用多选下拉。
- `basic.parallelModelIds` 表示全部快捷输入首轮默认追加的并行模型。
- `quickInputs[].parallelModelIds` 表示当前快捷输入在全局并行模型基础上额外追加的模型，保存前会去重。
- Jina 响应模板默认使用 `{{content}}` 占位符包裹原始 Reader 响应；如果模板里不包含该占位符，系统会把原始响应追加到模板尾部。
- 语言和主题切换都支持即时预览并在保存后持久化。
- 云同步分栏当前交付 `enabled / provider / gist / webdav / 测试连接 / 立即同步 / 最近同步时间`。
- 手动同步当前只推送配置快照，不拉取远端，不合并页面、会话和墓碑。
- 顶部“保存并同步”会先走保存链路，再走现有手动同步链路；“云同步”分栏仍保留“立即同步”。
- 导入配置改为文件选择与文件读取，不再使用粘贴 JSON 的临时输入方式。
- 黑名单分栏当前交付规则列表、新增、编辑、软删除、恢复默认规则和匹配测试。
- 设置页当前局部交互约束：
  - 行内动作优先使用 icon button，不再依赖原生 `title`。
  - 软删除动作统一使用按钮旁侧 mini confirm。
- 设置页操作错误改为顶部 toast，不再在主内容区单独铺一块错误面板。
- 内置默认黑名单当前覆盖 Google / Bing / 百度搜索结果页，旧配置同样采用“补缺不覆盖”。
- 规则恢复默认只恢复系统内置规则，不删除用户自定义规则。
- 基础设置展示当前本地缓存占用，并允许清理可安全回收的本地缓存数据；当前展示总保存页面数与智能格式化后的总大小。
- 当前已交付本地设置闭环与最小同步闭环，远端合并、删除传播和自动同步留到后续阶段单独实现。

## 6. 错误与异常处理

- 保存失败时保留当前表单。
- 导入失败时不覆盖现有配置。
- 非法黑名单正则在保存或导入阶段直接阻断，不把错误带到运行时。
- 导出失败时保留当前页面状态，并通过顶部 toast 提示错误。
- 本地缓存清理失败时保留当前占用统计并通过顶部 toast 提示错误，不影响其他配置编辑。
- 连接测试失败时保留当前同步表单并展示错误反馈。
- 手动同步失败时保留当前草稿和最近一次成功同步时间，不把失败结果写成成功状态。
- “保存并同步”中的保存失败会直接中断同步阶段。
- 当前不提供远端拉取或冲突合并，不承诺页面和会话数据已进入同步闭环。

## 7. 数据与状态

- UI 运行态：
  - 当前导航分栏
  - 已保存配置快照
  - 当前草稿配置
  - 当前展开的模型项
  - 本地缓存占用统计
  - 未保存变更标记
  - 快捷输入远端模板导入进行态
  - 同步测试进行态
  - 同步执行进行态
  - 最近一次同步反馈
  - 顶部 toast 提示
- 持久化依赖：
  - `ExtensionConfig`
  - `SyncSnapshot.lastSyncAt`

## 9. 约束与禁止事项

- 左侧栏目切换不能丢未保存表单。
- 左侧栏目数量和顺序必须与产品稿一致，不在导航区单独增加“导入导出”栏目。
- 设置页必须保持完整 tab 页面观感，不能回退成弹窗式整页包裹卡片。
- 默认模型候选只允许“启用且配置完整”的模型。
- “启用且配置完整”的模型定义在设置页、侧边栏和对话管理页必须一致。
- 图片输入能力必须通过 `supportsImages` 显式建模，不允许由 UI 或调度层按模型名猜测。
- Provider 切换后，表单字段显隐和校验规则必须同步切换，不能保留上一个 Provider 的无效必填约束。
- `TEST_SYNC_CONNECTION` 与 `SYNC_NOW` 必须走 background 命令链路，不允许 options 页面直接发远端请求。
- `SYNC_NOW` 必须先保存当前配置，再把成功结果回写到 `sync.lastSyncAt`。
- 黑名单表单只允许编辑草稿，真正持久化必须通过顶部保存或保存并同步走 background 命令链路。
- 国际化只允许使用平铺 key。
- 图标资源不能通过在线字体加载。
- 阶段2导出动作必须把配置内容真正交给用户，而不是只调用后台命令。
- 恢复默认必须通过 background 命令链路执行，不能由 UI 本地拼装默认值代替真实写回。

## 10. 测试要求

- 职责测试：模型、快捷输入编辑、快捷输入远端模板导入、语言切换、主题切换、本地缓存、恢复默认、文件导入导出、同步连接测试、保存并同步与手动同步、顶部 toast 错误反馈。
- 职责测试：本地缓存占用展示与缓存清理。
- 边界测试：不完整模型、重复 ID、非法黑名单正则、内置规则恢复。
- 正常流测试：保存、保存并同步、恢复默认、语言切换、主题切换、模型新增、复制、拖拽排序、软删除、展开编辑、快捷输入新增、编辑、折叠展开、拖拽排序、远端模板导入、软删除、导出配置、文件导入恢复、本地缓存清理、同步连接测试、手动同步、黑名单规则测试。
- 错误流测试：保存失败、导入失败、导出失败、缓存清理失败、同步连接失败、手动同步失败、保存并同步中的保存失败、background 命令返回业务错误。
- 异常流测试：快速重复保存、切换栏目后保持表单、Provider 切换后字段显隐恢复正确、service worker 启动后再注入测试 provider 仍可命中同步桩。
- 不变量测试：软删除配置项不破坏历史引用。
- 可观测性测试：错误 toast、API Key 掩码状态展示。
