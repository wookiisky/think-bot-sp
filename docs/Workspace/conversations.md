# 对话管理页

## 1. 模块定位

对话管理页是历史页面和历史会话的工作台，不是归档查看器。它负责按页面维度恢复上下文并继续工作。

## 2. 界面布局映射

对话管理页界面按设计图拆成 3 个稳定区域：

- 左侧历史栏：
  - 固定包含搜索框、历史页面列表和空状态。
  - 支持高密度列表浏览，列表项按最近更新时间排序。
  - 单条页面项固定包含网站图标、标题、打开原网页按钮和删除按钮。
  - 列表项局部动作统一使用 icon button，避免历史列表在窄宽度下被文字按钮挤压。
  - 打开与删除动作当前统一通过即时 tooltip 表达文案，其中删除动作必须走 mini confirm。
- 中间分隔条：
  - 支持左右栏宽度拖拽。
  - 只负责布局尺寸调整，不承载业务动作。
- 右侧工作区：
  - 顶部是页面标题和 URL 信息区，标题支持内联编辑。
  - 标题区下方是提取内容区，承担历史正文阅读职责。
  - 提取内容区下方是快捷输入标签区和聊天区。
  - 底部是继续对话输入区，包含图片预览、模型选择和发送动作。
  - 标签、消息和分支状态统一复用侧边栏的状态 icon 语义，不单独维护一套文字状态带。

## 3. 核心抽象

- 历史页面列表
- 选中页面上下文
- 提取内容区
- 恢复后的标签和聊天区
- 图片预览区
- 继续对话输入区

## 4. 能力边界

负责：

- 加载历史页面列表。
- 搜索、切换、删除页面。
- 恢复页面标题、URL、提取内容、标签和聊天。
- 编辑页面标题。
- 打开原网页、复制提取内容。
- 展示并切换快捷输入标签。
- 打开历史页面时只恢复快捷输入标签状态，不在页面恢复阶段自动触发发送。
- 展示当前选中主分支、其他分支结果和 Markdown 内容。
- 以响应式并行多列浏览同一助手消息的多分支，统一使用 `minmax(350px, 1fr)` 的 grid 布局。
- 在历史上下文上继续对话，支持图片输入、模型切换、停止、清空和导出。
- 恢复页面级和标签级 loading 状态。
- 复用侧边栏的 `ChatThread / ChatInput / WorkspaceStatusGlyph / workspace-copy`，保证文案和交互语义一致。
- 与侧边栏共享同一套消息操作：用户消息编辑/重试/复制，助手分支重试/复制/定位/分支操作，以及当前轮主分支切换。

不负责：

- 重新定义聊天协议。
- 直接修改同步远端。
- 直接执行页面 DOM 提取。

## 5. 对外接口

- 获取页面列表。
- 搜索页面。
- 获取单页面全部恢复数据。
- 编辑标题。
- 删除页面。
- 发送消息、停止会话、清空当前标签、编辑消息、用户消息重试、助手分支重试、切换主分支、复制消息、继续新增分支、停止分支、删除分支、导出。
- 订阅流式 chunk、完成、错误和 loading 更新。

## 6. 关键流程

- 首次加载历史页面列表。
- 搜索过滤。
- 选中页面恢复完整上下文。
- 拖拽调整左右栏宽度。
- 标题编辑并同步左侧列表。
- 复制提取内容和在新标签页打开原网页。
- 历史会话上继续提问，支持图片输入和模型切换。
- 停止当前标签会话、清空当前标签会话、导出当前标签会话。
- 恢复主会话与分支 loading。
- 多分支响应式并行多列布局。
- 删除页面并按同步状态选择软删或硬删。
- 页面删除和标签清空当前都复用与侧边栏一致的 mini confirm 交互，不使用原生 `window.confirm`。

## 7. 错误与异常处理

- 页面记录损坏时只影响当前右侧，不影响其他页面浏览。
- 标题保存失败时回退旧标题。
- 删除失败时不提前清空右侧视图。
- loading 恢复失败时降级为静态历史视图，不阻断其他页面切换。
- provider 返回明确错误文本时，直接显示该文本，不再用统一助手失败文案覆盖。

## 8. 数据与状态

- UI 运行态：
  - 当前搜索词
  - 当前选中页面
  - 左右栏宽度
  - 列表滚动位置
  - 标题编辑态
  - 当前活动标签
  - 右侧页面恢复状态
  - 提取内容区滚动位置
  - 聊天区滚动位置
  - 输入框草稿
  - 当前选中模型
  - 本地图片预览
  - 分支预览层显示状态
- 持久化依赖：
  - `PageRecord`
  - `ConversationRecord`
  - `LoadingStateRecord`
  - `ExtensionConfig`

页面级动作与标签级动作必须严格分离：

- 页面级动作：
  - 选中页面
  - 编辑标题
  - 复制提取内容
  - 打开原网页
  - 删除页面
- 标签级动作：
  - 切换标签
  - 发送消息
  - 停止当前标签会话
  - 清空当前标签会话
  - 导出当前标签会话
  - 编辑消息
  - 管理分支

## 10. 约束与禁止事项

- 左侧列表必须以页面元数据为 source of truth。
- 左侧列表项固定展示图标、标题、打开按钮和删除按钮，不能把关键动作折叠到二级入口。
- 删除按钮不能顺带触发页面选中。
- 打开原网页只开新标签，不改变右侧当前选中页面。
- 右侧必须同时呈现提取内容区和聊天区，不能把提取内容退化为某个标签的临时消息。
- 快捷输入标签必须保持高密度、多行换行布局，不能要求横向滚动访问常用标签。
- 右侧输入区行为要与侧边栏保持一致，包括模型选择、图片输入、停止、清空、导出和分支交互。
- 右侧输入区键盘协议要与侧边栏保持一致：`Enter` 发送、`Shift + Enter` 换行、composition 不发送。
- 右侧工具栏和消息局部动作默认使用 icon button，不把历史页做成独立的文字按钮体系。
- 左右栏拖拽只影响布局宽度，不改变当前页面、标签、输入草稿和滚动上下文。
- 继续对话行为要与侧边栏保持一致。
- 快捷输入标签行为要保持明确分层：
  - 页面恢复阶段只恢复历史与状态，不自动触发 quick input。
  - 用户手动点击空的 quick input 标签时，仍按侧边栏语义立即触发一次发送。
- 分支阅读列布局要与侧边栏保持一致，不能在历史页单独定义另一套分支浏览心智。

## 11. 测试要求

- 职责测试：列表、搜索、恢复、继续对话、标题编辑、删除。
- 职责测试：分隔条拖拽、标签切换、图片预览、模型切换、停止、清空、导出。
- 职责测试：分支横向阅读列与窄屏回落。
- 正常流测试：恢复已有页面并继续提问、编辑消息、继续新增分支。
- 正常流测试：打开历史页面不会自动发送 quick input，手动点击空 quick input 标签仍会发送。
- 正常流测试：复制提取内容、打开原网页不改变当前选中。
- 正常流测试：标题 `Enter` 保存、`Esc` 取消、失焦保存。
- 边界测试：空状态、无搜索结果、损坏页面数据、当前选中页面被删除。
- 错误流测试：标题保存失败、删除失败。
- 异常流测试：有 loading 状态时恢复页面，主会话和分支状态都可见。
- 删除语义测试：同步开启时软删、未开启时硬删。
- 不变量测试：删除页面不影响其他页面记录。
- 不变量测试：左右栏拖拽和打开原网页不影响当前右侧上下文。
- 可观测性测试：列表更新时间、loading 状态和错误状态清晰可见。

- 已落地真实历史工作台，不再使用 `PageShell` 占位页。
- 已落地页面列表、搜索、标题编辑、删除、详情恢复、继续对话与 loading 恢复。
- 已复用 side panel 的聊天命令、流式事件协议、`ChatThread` 和 `ChatInput`。
- 已复用共享工作台翻译函数和状态图标，历史页与侧边栏的 `promptTab` 文案、按钮可访问名称和 loading 视觉保持同源。
- 已复用共享 Markdown 渲染器和消息局部操作，对话管理页与侧边栏不再维护两套消息交互语义。
- 已复用共享分支多列布局：统一使用通用最小宽度和同一套主分支选择语义，不再让历史页维护独立分支容器。
- 已把历史正文区显示与复制统一收敛为“删除空行后的 Markdown 纯文本”，避免历史页与侧边栏的正文阅读密度不一致。
- 已把右侧 `promptTab` 改为与侧边栏一致的高密度装饰语义：有内容时显示与 tab 等宽的底部下划线，loading 继续使用边框流光。
- 已把右侧工作区收敛为固定布局：标题区、正文区、`promptTab` 区、notice 和输入区固定，消息区作为唯一纵向滚动容器。
- 已把助手消息整体切换为浅灰背景，保持与侧边栏一致的消息分层。
- 删除语义当前已接入：
  - 同步关闭：直接本地级联硬删。
  - 同步开启：先写入 page tombstone，再清理本地页面、会话与 loading。
- 草稿和图片预览仍保持页面内运行态，不在 conversations 关闭后持久化恢复。
